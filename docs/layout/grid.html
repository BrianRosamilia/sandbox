Description

Sandbox provides a float based responsive grid system. Only two ingredients are required to build a responsive layout: a grid, and grid columns within it.

All that div.grid actually is, is a block level element that has clearfix code applied to it (to ensure that the floated elements within it don't exceed the grid's boundaries). If you want to use some other block level element to house the grid columns, just apply the .clearfix class to it.

The grid columns are block level elements with some padding (set with the $grid-column-padding variable), and a set width which are floated to the left. The width of a grid column is a percentage based on the [#] you used.

12 Unit Grid

Sandbox's grid is split into 12 equal units. Each [#] you plug in (from 1 - 12) sets the width property to a percentage based on that number out of 12. This width percentage is in relation to the parent element.

Pushing

div.push-[#], and div.push-[#]-[breakpoint] are used to push a grid column to the right by the [#] specified ([#] being from 1 - 12). Sandbox does this by setting the margin-left property to the relevant percentage.

Stacking and Wrapping

Floated elements, such as the grid columns, will stack one next to the other, so long as there's enough room for them to fit within that line. If there's not enough room for the floated element to fit, it will wrap onto the next line.

Manual new-line

Sometimes floated elements misbehave, and don't go onto a new line when we expect them to. This may occur due to differing grid column heights. To remedy this, use div.new-line to force everything beneath it to start on a new line.

Default

As the name implies, div.unit-[#] can be used to specify a grid column width that isn't bound to a breakpoint. A breakpoint-specific grid column class may still be applied, and will only be active during that breakpoint.

unit-9-md unit-12

  1. <div class="grid">
    1. <div class="unit-12 unit-9-md">
      1. <!-- content -->
    2. </div>
  2. </div>

Nesting

Grid column classes can be nested directly within each other. Remember that grid column widths are percentage based in relation to the parent element, and also that grid column padding will stack up.

Note that in this example the color is being applied to the grid column itself, rather than its content area.

  1. <div class="grid">
    1. <div class="unit-12">
      1. <div class="unit-12-sm unit-10-md unit-8-lg">
        1. <div class="unit-4-sm unit-2-md unit-6-lg"></div>
        2. <div class="unit-4-sm unit-8-md unit-3-lg"></div>
        3. <div class="unit-4-sm unit-2-md unit-3-lg"></div>
      2. </div>
      3. <div class="unit-12-sm unit-10-md unit-8-lg">
        1. <div class="unit-4-sm unit-5-md unit-4-lg"></div>
        2. <div class="unit-4-sm unit-2-md unit-6-lg"></div>
        3. <div class="unit-4-sm unit-5-md unit-2-lg"></div>
      4. </div>
    2. </div>
  2. </div>
