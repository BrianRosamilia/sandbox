Description

The z-index classes come into play when working with an element that has had its position property set (to something other than static). The z-index property will have no effect on static or non-positioned elements.

The general idea is that positioned elements with a higher z-index value will appear above other positioned elements with a lower z-index value (or ones that don't have a z-index set). This also depends on the stacking context and source order that the element is in.

Typically an element that has a higher source order (higher up in the html) will appear above an element with a lower source order (lower down in the html).

If two positioned elements from different stacking contexts overlap eachother (let's say that b overlaps a), setting the z-index on a to 9000 won't have the desired effect of putting it on top. You must go to the beginning of a's stacking context and set the z-index there (this is usually where the position property was first set).

For a more in depth look at the z-index property, check out this article: Understanding CSS z-index
