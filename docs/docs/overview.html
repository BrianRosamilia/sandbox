Overview

Structure

The structure of this documentation resembles Sandbox's actual directory structure. Each page (except for this one) relates to a file in Sandbox, and contains information pertaining to that file.

Breakpoints

Sandbox provides three responsive breakpoints: small, medium, large. The table shows when each breakpoint becomes active, based on the width of the current device or browser.

Breakpoint rem px
small <= 26.25rem <= 420px
medium >= 26.3125rem and <= 48rem >= 421px and <= 768px
large >= 48.0625rem >= 769px

CSS Selector Syntax

This documentation uses CSS selector syntax to describe the classes and elements used. For example:

div means a <div> element.

.blue means the blue class

div.blue means a <div class="blue">

div.blue.round means a <div class="blue round">

input[type="password"] means <input type="password">

input[type="password"].blue means <input type="password" class="blue">

CSS 101

The following are some basic terms and ideas that will be used through the documentation.

Display Property

block level elements start on a new line, and take up the full width available.

inline level elements do not start on a new line, and only take up as much room as they need.

inline-block level elements, are formatted as blocks on the inside, and inline on the outside.

Rem and Em

rem is a relative unit that can be thought of as a multiplier of the font-size of the root element (the html element). Assuming the font-size of the html element is 16px, 1rem would be equal to 16px.

em is a relative unit that can also be thought of as a multiplier of the font-size of the current element itself, or the font-size of its most recently set ancestor (if the element itself didn't have a font-size set). If the element's font-size is set to 24px, 1em would be equal to 24px.

Position Property

static is the default setting for the position property. A static positioned element is included in the page flow. Using the top, bottom, left, right properties will have no effect on a static positioned element.

relative means that an element's position can be adjusted in relation to its normal position by using the top, bottom, left, and right properties. A relative positioned element is included in the page flow.

absolute means that an element can be positioned in relation to its most recently positioned ancestor (a static positioned ancestor does not count), by using the top, bottom, left, right properties. An absolute positioned element is removed from the page flow.

fixed means that an element can be positioned in relation to the viewport by using the top, bottom, left, and right properties. A fixed positioned element is removed from normal page flow.
